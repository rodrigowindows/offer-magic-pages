# Guia de Organização da Documentação

Este documento explica a estrutura da documentação, onde encontrar cada tipo de informação e como contribuir.

## Estrutura de Pastas

- **getting-started/**: Guias rápidos e de início
- **features/**: Documentação consolidada de features principais (ATTOM/AVM, COMPS, AB Testing, CSV Import, Migração)
- **deployment/**: Guias e instruções de deploy
- **testing/**: Testes, validações e checklists
- **troubleshooting/**: Problemas comuns e soluções
- **archive/**: Arquivos antigos, históricos e versões anteriores

## Como adicionar nova documentação

1. Escolha a categoria correta (feature, deploy, testing, etc.)
2. Crie ou edite o arquivo na pasta correspondente
3. Atualize o índice em docs/README.md e docs/INDEX.md
4. Se for um histórico, mova versões antigas para archive/

## Dicas
- Sempre consolide informações duplicadas
- Use títulos claros e objetivos
- Mantenha os links funcionando
